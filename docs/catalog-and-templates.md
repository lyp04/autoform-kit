# 表单目录与表单模板（概念说明）

本页解释「表单目录（catalog）」和「表单模板（form profile）」是什么，以及如何在面板网页里创建、编辑、排序和发布它们。

> **推荐始终通过面板 UI 管理，不要手改仓库里的 JSON。** 面板会替你处理底层数据结构与校验。

---

## 表单目录（Catalog）

- 目录是「一组表单模板」的**有序集合**，存放在你自己的 GitHub 仓库里，由面板统一管理（读写都通过面板完成）。
- App 登录并同步后，会**按目录里的顺序**把可用表单展示给操作员。

## 表单模板（Form profile）

- 一个表单模板描述「一张表单」：它的标识（如 id / 名称）、操作员要完成的若干步骤 / 字段（例如文本输入、拍照步骤等），以及用于显示的标签（可含多语言）。
- 一个目录里可以有多个表单模板，分别对应不同的产品 / 工序 / 场景。

## 职责边界与敏感配置

- App 只负责通用的扫码、拍照、字段渲染、提交和错误处理。
- Panel / catalog profile 负责具体的 `template`、`gradeMap`、`previousStepTemplates`、`autoCreatePreviousSteps` 等业务配置。
- 真实产品名、SKU、模板 ID、仓库 ID 等应留在你的私有目录中，不要提交到公开源码或文档。

### 前置步骤模板

如果某表单需要补录前置步骤，请在面板的表单设置中填写第 1 / 2 步的**精确模板 ID**。两个 ID 必须都是正整数，且详情接口返回的步骤号应与配置一致。App 不根据相邻 ID、产品名、SKU 或 SN 相似度猜测模板。

从旧版本升级时，请先为需要此功能的 profile 补齐 `previousStepTemplates.step1TemplateId` 和 `step2TemplateId`，再在面板显式启用自动补录并至少选一个等级。对应 JSON 必须是 `autoCreatePreviousSteps: { "enabled": true, "grades": ["A"] }` 这类明确配置；只有旧 `gradeASpecialHandling` 标记、只有模板 ID，或缺少 `grades` 都不会触发。没有精确配置时，App 会明确报错并停止，不会选择一个“看起来相似”的模板。

这与“预检后人工确认修复”是两条不同路径：非 A 等级连续预检确认前置步骤确实缺失后，操作员继续并手动拍摄运行图，App 可以执行一次显式修复；它不是按等级隐式开启的自动补录。两条路径都只从 `previousStepTemplates` 读取精确 ID，缺少映射时都会明确报错。

## 在面板里管理

在面板 UI 里你可以：

- **新建（添加新机型）**：面板**不是**让你从零手写表单，而是走这样一条路径——点「➕ 添加新机型」→ 按关键字搜索**你后端的模板列表** → 选中某个模板后，面板会**按后端真实模板自动生成一份草稿 profile** → 你再用「✨ AI 帮我改」的大白话文本框、或「高级：直接看 / 改 JSON」的结构化 / JSON 编辑器细调 → 发布。
  > **前置条件**：这条路径依赖你的**后端提供「模板列表 / 模板详情」接口**；面板正是从这里拉取模板来生成草稿的。
- **编辑**：修改已有模板（同样通过预览 + AI 文本框 / JSON 编辑器）。
- **排序**：在顶部 **⚙ 全局设置** 对话框的「**表单顺序**」里，用 **▲ / ▼** 按钮调整机型的先后顺序（**不是拖拽**），点「保存顺序」生效 —— 这个顺序**就是** App 端表单列表的显示顺序。
- **发布**：把改动写回目录仓库；App 下次自动同步即可看到。

> 请不要绕过 UI 直接编辑仓库里的 JSON —— 面板 UI 才是管理表单模板的推荐方式。

## AI 标签翻译 / 优化（可选）

如果部署时配置了 `AI_API_KEY`（见 [worker-setup.md](./worker-setup.md)），发布时面板可以：

- **翻译**：把标签自动翻译成 App 支持的多种语言（如 `zh` / `en` / `es`）。
- **精简**：把过长的标签缩短，便于在设备小屏上显示。

此功能是**可选**的；不配置 AI Key 时，你也可以手动填写各语言的标签。

## 一个「仅示意」的例子

下面只是一个**极简、纯属虚构**的示意，用来帮助理解「模板里大概有什么」。**这不是真实的字段结构**，实际结构请以面板 UI 为准。

```jsonc
// 仅示意（illustrative only）—— 并非真实 schema。请用面板 UI 管理模板。
{
  "id": "sample-widget-a",
  "title": "Sample Widget A",
  "steps": [
    { "type": "text",  "label": "Serial number" },
    { "type": "photo", "label": "Top-side photo" }
  ]
}
```

真实模板包含的字段更多、含义也更丰富。请通过面板 UI 创建和编辑，**不要照抄上面的结构**。
