# 把 App 连接到你的面板

在 App 里完成以下设置，即可让它从你的面板拉取表单，并把录入结果提交到你的后端。

---

## 1. 在 App 设置里填写面板信息

打开 App 的「设置」，填写：

- **面板地址（Panel URL）**：你部署的 Worker 地址，例如 `https://your-panel.your-subdomain.workers.dev`
- **访问密钥（Access key）**：与面板 `CATALOG_READ_KEY` **相同**的值（见 [worker-setup.md](./worker-setup.md)）

填好后点击**保存**。

## 2. 登录后端

用你的后端账号登录：

- 账号 + 密码（+ 图形验证码，如后端要求）
- 登录成功后，App 会持有一个登录态（token），用于提交录入并被面板校验。

## 3. 表单自动同步（无需手动操作）

App **没有「同步」按钮**——目录同步是**自动**的，会在以下时机发生：

- App **启动**时；
- App **切回前台**时（有限流，约每 10 分钟最多一次）；
- 你在第 1 步**保存**面板地址 / 访问密钥之后，立即同步一次。

同步时 App 会用访问密钥从面板拉取表单目录，并**按目录顺序**显示可用表单。因此填好第 1 步并保存后，表单通常就已经拉好了，直接开始扫码 / 拍照 / 填写并提交即可。

---

## 多语言切换

App 界面支持多语言（`zh` / `en` / `es`）。在设置里切换语言即可；若目录里为标签配置了多语言，表单标签也会随界面语言切换。

## 自更新渠道

App 可从一个**公开的** GitHub 仓库检查并下载新版本 APK（因为是公开仓库，所以**不需要 token**）。这个仓库由面板的全局设置指定（见 [configuration.md](./configuration.md)）。在 App 内触发「检查更新」即可获取最新正式版。

---

## 常见问题

- **同步失败 / 提示无权限**：多半是**面板地址**或**访问密钥**填错，核对后重试。
- **登录失败**：确认后端地址（面板全局设置里的 *Backend API base*）正确、账号密码 / 验证码无误。
- **看不到表单**：确认目录里已有**已发布**的表单，且面板地址、访问密钥均正确。
